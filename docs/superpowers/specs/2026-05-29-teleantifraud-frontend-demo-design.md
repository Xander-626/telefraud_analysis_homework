# TeleAntiFraud 答辩前端演示方案设计

## 目标

本前端用于深度学习课程期末答辩现场展示 TeleAntiFraud 电诈检测模型的实际效果。PPT 负责讲解项目背景、模型路线和实验结论；前端负责完成可视化现场演示，让老师和同学直观看到“输入通话样例 -> 模型检测 -> 输出电诈风险”的闭环。

前端应优先保证稳定、直观、可控，而不是做成复杂业务系统。主演示路径使用预置样例，降低现场网络、显卡、模型加载、音频转码等不确定因素带来的风险；上传检测和文本检测作为增强入口保留。

## 推荐形态

前端命名为 **TeleAntiFraud 电诈检测演示工作台**，采用单页应用形式。打开页面后直接进入检测工作台，不制作营销式首页。

主流程：

1. 选择预置正常通话或电诈通话样例。
2. 点击“开始检测”。
3. 页面展示模型处理状态。
4. 输出 `正常通话` 或 `疑似电诈` 结论。
5. 展示 fraud probability、风险等级、ASR 文本、关键风险片段和模型流程。
6. 使用底部指标卡片说明模型在测试集上的效果。

## 信息架构

### 左侧输入区

输入区使用三个 Tab：

1. **预置样例**
   - 正常通话样例。
   - 电诈通话样例 1。
   - 电诈通话样例 2。
   - 每个样例展示标签、音频播放器、简短说明和“开始检测”按钮。
   - 答辩现场优先使用该入口。

2. **上传检测**
   - 支持 `.mp3` 和 `.wav`。
   - 展示文件名、文件大小、上传状态和检测按钮。
   - 标注“实时音频处理可能需要更长时间”，避免现场等待时显得像系统异常。
   - 上传检测作为增强能力，不作为答辩主路径依赖。

3. **文本检测**
   - 支持粘贴 ASR 文本或通话内容。
   - 用于老师临时提出文本片段时快速演示。
   - 结果应标注为“文本分支演示”，避免与完整音频+文本融合链路混淆。

### 右侧结果区

检测前展示模型流程：

`音频输入 -> Whisper-small ASR/音频特征 -> Chinese RoBERTa 文本特征 -> MLP 融合分类器 -> normal/fraud`

检测后展示：

- 结论徽标：`疑似电诈` 或 `正常通话`。
- 风险分数：`fraud probability`，保留 3 位小数即可。
- 风险等级条：低风险、中风险、高风险。
- ASR/输入文本：展示模型用于判断的通话内容。
- 风险依据：展示 2-4 个高风险关键词、话术片段或规则化解释。
- 模型路径说明：强调该项目使用音频和文本特征融合，而不是单纯关键词匹配。
- 操作按钮：重新检测、查看流程、复制结果摘要。

### 底部指标区

底部展示实验结果，不占用主操作空间。建议使用小卡片和对比表：

指标卡片：

- 测试集：400 条。
- Accuracy：1.000。
- Precision：1.000。
- Recall：1.000。
- F1：1.000。
- FP / FN：0 / 0。

方案对比表：

| 方案 | 最佳 F1 | 模型大小 | 训练时间 | 答辩结论 |
| --- | ---: | ---: | ---: | --- |
| Prompt + MLP | 1.000 | 1.6 MB | 约 5 min | 基线可用 |
| ASR + MLP | 1.000 | 1.6 MB | 约 5 min | 推荐部署 |
| ASR + E2E | 0.995 | 338 MB | 约 3 h | 性价比较低 |
| SFT/LoRA | smoke test | adapter | 较长 | 后续扩展 |

主展示模型应标注为 **Whisper-small ASR + Chinese RoBERTa + MLP Fusion Classifier**。SFT/LoRA 当前不作为主检测入口，只作为扩展路线说明。

## 交互状态

前端需要覆盖以下状态：

- 初始状态：展示输入区和模型流程。
- 样例选中：高亮当前样例，音频播放器可播放。
- 检测中：按钮进入 loading，结果区显示步骤进度。
- 检测成功：显示结论、风险分数、依据和摘要。
- 上传待检测：显示文件信息。
- 上传错误：提示格式不支持、文件过大或处理失败。
- 文本为空：禁止检测并提示输入内容。
- 后端不可用：展示兜底提示，并允许继续使用预置样例的离线结果。

## 后端接口

为保证答辩现场稳定，建议使用两层接口。

### 预置样例接口

`POST /api/demo/predict-sample`

请求：

```json
{
  "sample_id": "fraud_example_1"
}
```

响应：

```json
{
  "sample_id": "fraud_example_1",
  "prediction": "fraud",
  "fraud_probability": 0.982,
  "risk_level": "high",
  "asr_text": "这里展示该音频对应的转写或摘要文本",
  "evidence": ["高风险话术片段 1", "高风险话术片段 2"],
  "model": "Whisper-small ASR + Chinese RoBERTa + MLP Fusion Classifier"
}
```

该接口可直接读取预先计算好的样例结果，确保响应速度稳定。答辩主路径使用此接口。

### 实时检测接口

`POST /api/predict`

支持两种输入：

- `multipart/form-data` 上传音频。
- JSON 文本输入。

响应字段与预置样例接口保持一致。该接口作为增强能力，允许真实上传或现场文本检测。如果实时音频处理速度较慢，前端必须清楚展示进度和说明。

## 数据来源

前端展示的实验指标来自项目训练日志和 `runs` 目录中的 metrics：

- `runs/binary_fusion_whisper_small_roberta/metrics.json`
- `runs/binary_fusion_whisper_small_asr_roberta/metrics.json`
- `runs/binary_e2e_asr/metrics.json`
- `runs/sft_lora_smoke/metrics.json`

主推荐方案采用 `runs/binary_fusion_whisper_small_asr_roberta` 的 ASR + MLP 结果。它在多个 epoch 达到 F1=1.0，并且模型体积小、训练快，更适合课程答辩现场展示。

## 视觉风格

整体风格应接近“安全风控控制台”：

- 信息清晰、层级明确。
- 投影环境下字号足够大。
- 主色建议使用蓝色或青色表达分析系统感。
- 红色只用于高风险结论、风险片段和告警状态。
- 避免大面积装饰图和复杂动画。
- 第一屏必须直接呈现可操作检测界面。

## 答辩演示脚本

推荐现场演示顺序：

1. 打开工作台，说明这是电诈检测模型的前端演示界面。
2. 选择正常通话样例，点击开始检测。
3. 展示输出为正常通话，风险分数较低。
4. 选择电诈通话样例，点击开始检测。
5. 展示输出为疑似电诈，风险分数较高，并指出 ASR 文本和风险片段。
6. 简要说明模型流程：Whisper 提取音频/转写信息，RoBERTa 编码文本语义，MLP 完成融合分类。
7. 指向底部指标区，说明测试集 400 条，推荐方案达到 F1=1.0、FP=0、FN=0。
8. 如老师提问，可切换到文本检测或上传检测入口做补充展示。

## 范围边界

本方案只设计前端演示体验及其所需接口，不包含 PPT 制作。

本方案不要求现场全量运行 SFT/LoRA，也不把 SFT/LoRA 作为主检测路径。SFT/LoRA 当前应展示为后续扩展方向，避免答辩现场把未完成的实验包装成主结果。

本方案不要求开发完整用户系统、登录权限、历史记录管理或复杂数据后台。这些功能会分散答辩重点，不符合期末课程展示的优先级。

## 验收标准

- 页面打开后第一屏就是检测工作台。
- 至少包含 1 条正常样例和 2 条电诈样例。
- 预置样例检测可在 1 秒内返回结果。
- 检测结果必须包含预测标签、fraud probability、风险等级和模型依据。
- 页面必须展示模型流程和核心实验指标。
- 后端不可用时，预置样例仍可通过离线结果完成演示。
- 上传检测和文本检测入口存在，但不影响预置样例主路径。
